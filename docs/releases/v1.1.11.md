# Skill Repo Tracker v1.1.11

## 中文

v1.1.11 让 GitHub 检测、Skill 更新和应用版本显示更可信。它重点解决“明明配置了 token 却像匿名请求一样被 60 次配额挡住”、“更新后状态看起来成功但再次检测又可更新”、以及“关于页版本滞后”这类影响判断的问题。

### 更新亮点

- GitHub 检测现在会优先使用仓库绑定的账号；仓库没有绑定账号时，会自动使用已验证账号作为兜底，减少误落到匿名 60 次/小时配额的情况。
- 检测入口增加问号说明，解释未认证请求通常只有 60 次/小时，认证请求通常是 5,000 次/小时，并提示超限后需要等 `x-ratelimit-reset` 指定时间再请求。
- 任务日志会更清楚地区分 token 无效、权限不足、匿名配额耗尽、认证配额耗尽和 secondary rate limit，不再把所有 GitHub 拒绝请求都混成同一种原因。
- Skill 更新闭环更稳定：更新后再次检测来源时，会根据实际内容判断是否仍可更新，减少“按钮只改状态”的假成功感。
- 任务页的重试语义更明确，可安全重试的任务才会真正重放原动作；不能安全重试的历史任务会显示不可重试原因。
- 设置页关于信息改为读取应用实际版本，避免安装了新包但界面仍显示旧版本。
- 项目协作、发布素材、本地验证包和正式发版规则已固化到 `AGENTS.md`，后续发布更一致。

## English

v1.1.11 makes GitHub checks, Skill updates, and app version reporting more
trustworthy. It focuses on issues that make the app hard to reason about: a
configured token still behaving like an anonymous 60-requests-per-hour check,
an update looking successful but becoming update-available again after a scan,
and the About panel showing an older version.

### Highlights

- GitHub checks now prefer the account bound to a repository. If a repository
  is not bound to an account, the app falls back to a verified account, reducing
  accidental anonymous requests.
- Check entry points now include help text explaining that unauthenticated
  requests are usually limited to 60 per hour, while authenticated requests are
  usually limited to 5,000 per hour. When a limit is exhausted, wait until the
  `x-ratelimit-reset` time before trying again.
- Task logs now separate invalid tokens, missing permissions, anonymous rate
  limits, authenticated rate limits, and secondary rate limits, instead of
  grouping every GitHub refusal into one vague error.
- Skill updates now close the loop more reliably: after an update, source
  detection compares actual content before reporting that the Skill is still
  update-available.
- Task retry behavior is clearer. Retryable tasks replay the original action;
  older or unsafe tasks show why they cannot be retried.
- The Settings About panel now reads the actual app version from the backend,
  preventing stale UI version labels after installing a new build.
- Project collaboration, local build, artifact, and formal release rules are
  now recorded in `AGENTS.md` so future releases stay consistent.

- Local build file: `Skill Repo Tracker_1.1.11_aarch64.dmg`

Install notes:

1. Download and open `Skill.Repo.Tracker_1.1.11_aarch64.dmg`.
2. Drag `Skill Repo Tracker.app` into `/Applications`.
3. On first launch, macOS may block the app because it is ad-hoc signed. Control-click the app in Finder, choose Open, then confirm Open.

5. Technical users can clear the quarantine attribute:

```bash
xattr -cr "/Applications/Skill Repo Tracker.app"
```

Safety boundaries:

- GitHub tokens are never included in migration packages. Add your token again
  on each new machine.
- This DMG is an ad-hoc test package, not a Developer ID notarized public
  installer.
